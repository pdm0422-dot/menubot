# 식당 메뉴 자동 카톡 발송 봇

매일 08:50(한국시간)에 식당 3곳의 카카오톡 채널 '소식'을 확인해서,
새 메뉴 사진이 올라왔으면 본인 카톡 '나와의 채팅'으로 사진+식당이름+링크를 보내주고,
아직 안 올라왔으면 "아직 안 올라왔어요" 알림만 보내줍니다.

## 1. GitHub 저장소 만들기

1. github.com 가입 (이미 있으면 생략)
2. 새 저장소(Repository) 만들기 → 이름 예: `menu-bot` → Private로 설정 추천
3. 이 폴더 안의 모든 파일을 그 저장소에 업로드
   - GitHub 웹사이트에서 "Add file > Upload files"로 통째로 드래그해서 올리면 됩니다

## 2. GitHub Secrets 등록 (중요 — 토큰을 코드에 직접 넣지 않기 위함)

저장소 페이지에서:

1. **Settings** 탭 클릭
2. 왼쪽 메뉴 **Secrets and variables > Actions** 클릭
3. **New repository secret** 클릭해서 아래 2개를 각각 등록:

| 이름 | 값 |
|---|---|
| `KAKAO_REST_API_KEY` | 카카오 디벨로퍼스에서 발급받은 REST API 키 |
| `KAKAO_REFRESH_TOKEN` | 이전에 발급받은 refresh_token 값 |

(access_token이 아니라 **refresh_token**을 넣어야 해요 — 매번 실행할 때마다 이 refresh_token으로
새 access_token을 자동으로 발급받는 구조예요)

## 3. 첫 실행은 '디버그 모드'로 직접 테스트

화면 구조를 제가 직접 못 봤기 때문에, 처음엔 수동으로 한 번 실행해서 잘 동작하는지 확인이 필요해요.

1. 저장소의 **Actions** 탭 클릭
2. 왼쪽에서 "식당 메뉴 카톡 발송" 워크플로 클릭
3. 오른쪽 **Run workflow** 버튼 클릭
4. `debug` 항목을 `true`로 선택하고 실행
5. 실행이 끝나면 결과 페이지 맨 아래 **Artifacts**에서 `debug-screenshots`를 다운로드 →
   각 식당 페이지가 실제로 어떻게 보였는지, 어떤 사진을 찾았는지 확인 가능

6. 본인 카톡 '나와의 채팅'에도 메시지가 왔는지 확인해주세요

**만약 이미지를 못 찾았다는 메시지가 오거나 스크린샷이 이상하면, 그 스크린샷을 저한테 보여주세요.
페이지 구조에 맞춰 `scraper.py`를 같이 조정하면 돼요.**

## 4. 정상 동작 확인되면 끝!

이후엔 매일 08:50(KST)에 자동으로 실행돼요. 따로 할 일 없어요.

## 5. 알아두면 좋은 점

- **refresh_token 유효기간**: 보통 약 2개월이에요. 만료되면 자동 발송이 멈추고
  Actions 탭에 실패 표시가 떠요. 그때는 처음에 했던 "5단계: 인증 URL → code → 토큰 교환"
  과정을 한 번 더 해서 새 refresh_token으로 Secret 값을 교체해주면 돼요. (다시 도와드릴게요)
- **state.json**: 어제 보낸 사진을 기억해서 "새 메뉴인지 아닌지" 판단하는 파일이에요.
  자동으로 갱신되니 따로 건드릴 필요 없어요.
- 식당이 추가/변경되면 `config.json`만 수정하면 돼요.

## 파일 구성

```
menu-bot/
├── config.json              # 식당 목록 (이름 + 채널 주소)
├── state.json                # 마지막으로 보낸 사진 기록 (자동 갱신)
├── requirements.txt          # 필요한 파이썬 패키지
├── scraper.py                # 채널 페이지에서 최신 사진 추출
├── kakao_send.py              # 카카오 '나에게 메시지 보내기' 발송
├── main.py                   # 전체 흐름 (매일 실행되는 메인 스크립트)
└── .github/workflows/
    └── menu-bot.yml            # 매일 08:50 자동 실행 설정
```
